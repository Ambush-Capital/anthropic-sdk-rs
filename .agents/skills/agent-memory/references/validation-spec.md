# Validation Specification

Specification for `check-agent-memory.mjs` — a validation script that runs as part of `just check` to verify memory system integrity.

## Scope

The script validates all initialized memory directories in the project: root and every subfolder with `docs/agent-memory/`.

## Checks

### Error-Level (must pass)

| Check | Rule |
|-------|------|
| INDEX exists | Every `docs/agent-memory/` directory has an `INDEX.md` |
| No orphans | Every `.md` file in `docs/agent-memory/` (excluding INDEX.md, known-pitfalls.md, and .scratch.md) has an entry in INDEX.md |
| Routing rules valid | Every file path in AGENTS.md routing rules points to an existing file |
| Topic size limit | No topic file exceeds 200 lines |
| Pitfall cap | Every `known-pitfalls.md` has <= 10 entries |
| No cross-level duplication | Subfolder topics do not repeat root topic content (checked by filename — same filename at multiple levels triggers a warning for manual review) |
| Frontmatter valid | Every topic file (excluding INDEX.md, known-pitfalls.md, .scratch.md) has valid YAML frontmatter with required fields: `scope`, `keywords`, `last-updated` |
| Scratch gitignored | `.gitignore` contains `**/.scratch.md` pattern |

### Warning-Level (flag, do not fail)

| Check | Rule |
|-------|------|
| Staleness | Flag topics where `last_updated` in INDEX.md is >21 days old |
| Undersized topics | Flag topic files <50 lines (consider merging) |
| INDEX size | Flag INDEX.md files >60 lines (approaching 80-line cap) |
| Managed block present | AGENTS.md at each initialized level has `<!-- managed:agent-memory:begin -->` markers |
| Scratch not empty | Flag if `.scratch.md` has content beyond the template (suggests previous session didn't triage) |

## Output Format

```
agent-memory check:
  ✓ root: INDEX.md exists
  ✓ root: no orphan files
  ✗ root: architecture.md exceeds 200 lines (213 lines)
  ⚠ root: conventions.md is stale (last updated 2026-01-15)
  ✓ src/api/: INDEX.md exists
  ...

Errors: 1  Warnings: 1  OK: 8
```

Exit code 0 if no errors. Exit code 1 if any error-level check fails.

## Integration

Add to `justfile`:

```
check-agent-memory:
    node scripts/check-agent-memory.mjs

check: check-agent-memory
```

The validation script is NOT part of the skill itself — it is a project-level tool generated by the `init` sub-command when requested. The skill should offer to create it during `init` but not require it.
